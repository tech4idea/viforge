# Third-Party Notices

This file records third-party components and redistributable binaries that matter for ViForge source and desktop releases. It is a release checklist aid, not a substitute for reviewing the exact dependency graph and bundled binary contents for each release.

## Source Dependencies

ViForge is a pnpm workspace using TypeScript, React, Vite, Hono, LangChain/LangGraph, Electron, electron-builder, esbuild, sharp, and related npm packages. Source releases should include the repository `LICENSE` file and preserve package-level license metadata from `package.json` and `pnpm-lock.yaml`.

Before a public release, generate and review an npm dependency license report for the exact lockfile used by the release. The release should not proceed if a dependency license is incompatible with the intended distribution model.

## Desktop Runtime Components

Desktop builds may redistribute these major runtime components:

| Component | Purpose | Notice requirement |
| --- | --- | --- |
| Electron | Desktop runtime and Chromium shell | Include Electron and Chromium license files generated by electron-builder in the release artifact. |
| Chromium | Browser engine bundled by Electron | Preserve Chromium third-party license notices generated by electron-builder. |
| PostgreSQL | Embedded local database for LangGraph checkpoint/store | Include PostgreSQL license/copyright text with the bundled PostgreSQL resources. |
| pgvector | PostgreSQL vector extension | Include pgvector license text when the extension is bundled. |
| Playwriter | Browser relay/CLI used by agent browser tools | Preserve package license metadata and notices for the bundled CLI code and dependencies. |

## PostgreSQL Bundle

PostgreSQL and pgvector binaries are not committed as normal source files. Release builds should either build them from documented source versions or consume a traceable binary bundle. The release artifact must make these facts auditable:

- PostgreSQL source version.
- pgvector source version.
- Build platform and build script or CI job.
- Runtime dynamic libraries copied into the bundle.
- License and notice files copied into the release artifact.

The desktop release guide documents the expected resource layout and build commands: `docs/current/19-desktop-release-guide.md`.

## Model Providers

ViForge does not redistribute model weights and does not include a hosted model service. Users configure their own OpenAI-compatible provider and are responsible for following that provider's terms.

## Browser Automation

ViForge uses Playwriter to connect to browser tabs explicitly authorized by the user. Release notes and user documentation must preserve the safety boundary: high-risk actions such as login, authorization, publishing, deletion, payments, purchases, or online configuration changes require user confirmation.
